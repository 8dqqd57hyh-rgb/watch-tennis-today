# Player Live Link Canonical Cleanup

## What changed

- Added `safeWatchPlayerLiveUrl()` and `watchPlayerLiveUrl()` to the canonical player utility.
- Replaced local slug generation for `/watch-player-live/...` links in daily/live components.
- Added middleware safety handling for clearly invalid `/watch-player-live/...` paths.
- Unknown or doubles-style player fragments are no longer promoted into player live hub links.

## Why

Production logs showed invalid player-style slugs were still generated by live/daily UI surfaces. This cleanup keeps the site focused on valid canonical player hubs and prevents match/doubles fragments from becoming indexable player destinations.
